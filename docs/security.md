# Security

The lab uses practical Kubernetes hardening without making the local workflow
too heavy.

## Implemented Controls

- Pod Security admission labels on the `stg` namespace in `restricted` mode.
- Dedicated ServiceAccounts with token automount disabled.
- Non-root API and Redis containers.
- Runtime default seccomp profiles.
- Privilege escalation disabled.
- Linux capabilities dropped.
- Read-only API root filesystem.
- Resource requests and limits on every container.
- ResourceQuota and LimitRange for the namespace.
- Default-deny NetworkPolicy with explicit DNS and API-to-Redis traffic.

## Secret Handling

`api-secrets` is generated by Kustomize with a lab-only token. Do not put real
secrets in this repository. For a real cluster, replace the generator with an
external secret manager, sealed secret, or CI-injected secret.

## NetworkPolicy Note

k3s network policy enforcement depends on the configured CNI and network policy
controller. The manifests declare the intended policy model; verify enforcement
in the target cluster before relying on it for isolation.
